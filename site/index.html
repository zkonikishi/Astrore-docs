Astrore

跨平台 Minecraft Java 服务端管理器

基于 Tauri 2 + Rust 构建，支持桌面端本地管理和浏览器远程控制。
内置 AI 运维助手，集成 35+ 模型厂商。

Agent 网页端

无需桌面环境，通过浏览器管理 Minecraft 服务端。

1

下载 Agent 一体包

从 GitHub Releases 下载对应平台的 Agent 包。Agent 与桌面应用共享 Astrore 版本号，便于确认是否同步更新。

2

解压并启动

Windows 双击 start-agent.bat，Linux/macOS 运行 ./start-agent.sh。

3

浏览器访问

打开 http://127.0.0.1:1421 进入网页控制台。

# 手动启动 Agent（自定义端口和令牌）
ASTRORE_BIND=0.0.0.0:1421 \
ASTRORE_TOKEN='your-secret-token' \
./astrore-agent

Agent 同时提供 HTTP API、WebSocket 实时推送和 Minecraft 服务端管理能力。访问 /health 可查看 Agent 版本；公网部署建议通过 Caddy / Nginx 反向代理启用 HTTPS。

使用指南

🚀 快速开始

  1. 下载并启动 Astrore
  2. 在「实例配置」中选择实例目录、Java 和服务端核心
  3. 配置内存和 JVM 参数
  4. 点击「启动服务端」
  5. 在「控制面板」查看实时日志并发送命令

⚙️ 配置服务端

  1. 进入「实例配置」顶部标签编辑 server.properties
  2. 调整游戏模式、难度、端口等
  3. 在应用内「扩展商店」中安装插件、模组和运维工具
  4. 在左侧「玩家权限」中设置 OP、白名单和封禁列表

🤖 AI 运维助手

  1. 在「控制面板」右侧使用 AI 助手
  2. 进入「软件设置」→「AI 助手设置」选择模型厂商
  3. 填入 API Key
  4. AI 会结合当前服务器状态和日志分析问题

🔄 崩溃自动重启

  1. 进入「实例配置」→「Paper / Purpur 优化设置」或「软件设置」中的运行策略
  2. 启用自动重启
  3. 设置最大重启次数和延迟
  4. 服务端异常退出时自动恢复
