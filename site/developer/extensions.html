扩展开发与发布

Astrore 使用 HTTPS JSON 注册表发现扩展，安装包为根目录包含 manifest.json 的 tar.gz 文件。

运行模式

WASI 是第三方扩展的推荐模式，将由 Astrore 沙箱限制文件、网络和系统访问。

external-mcp 用于 Python、Node.js、Rust 可执行文件等兼容扩展。它拥有当前用户的系统权限，因此始终标记为高风险，且不会静默启动。

权限规范

只声明真正需要的最小权限。推荐命名：

  • server.status.read
  • server.players.read
  • server.command.send
  • storage.extension-private
  • network:api.example.com

外部 MCP 进程无法完全沙箱化，即使没有声明权限，也会被视为高风险扩展。

发布和更新

三方作者只需要提供公开 Git 仓库链接用于审核。仓库应包含 manifest.json、源码文件和简短 README；官方维护者或自动化流程负责打包、计算 SHA-256 和文件大小，并写入可安装注册表。

  1. 确保安装包根目录包含 manifest.json。
  2. 将不可变的 tar.gz 上传至 GitHub Release 或其他 HTTPS 主机。
  3. 计算安装包的 SHA-256 和文件字节数。
  4. 提高语义版本号，并更新兼容注册表中的扩展条目。
  5. 向官方商店提交修改 registry/index.json 的 Pull Request。

Astrore 打开扩展商店时会自动读取已配置的注册表，比对已安装 manifest 版本并标记可更新扩展。更新时会重新执行 HTTPS、大小、SHA-256、安全路径和清单一致性校验，并在替换前备份旧版本。
